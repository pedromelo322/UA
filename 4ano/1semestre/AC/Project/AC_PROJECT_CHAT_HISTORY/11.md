A tua análise com o Wireshark foi perfeita e dá-nos a prova definitiva do que está a acontecer.

Repara com atenção no pacote **Lisbon-Core2** (o pacote que sai do router e onde o QoS devia atuar):

```text
MultiProtocol Label Switching Header, Label: 26, Exp: 3, S: 1, TTL: 62
Internet Protocol Version 4, ... Differentiated Services Field: 0x68 (DSCP: AF31 ...)

```

**A Prova:** O pacote sai como **MPLS** (tem cabeçalho Label). O Router fez o trabalho dele: pegou no **DSCP AF31** e traduziu automaticamente para **MPLS Exp: 3**.

**O Problema:** A tua configuração diz `match ip dscp af31`.
Mas, no momento em que o pacote sai da interface `FastEthernet0/1` (que tem `mpls ip` ativado), o router vê um pacote MPLS. O comando `match ip dscp` tenta olhar para o IP, mas o router (nesta plataforma 7200) não consegue "desembrulhar" o MPLS na saída para validar o IP. Por isso o contador fica a zero.

### A Solução "Híbrida" (Para funcionar e respeitar o Guião)

Para teres o projeto a funcionar (contadores a subir) e manteres a configuração "fiel" ao guião (que pede DSCP), a única solução técnica é usar o `match-any`.

Isto diz ao router: *"Se conseguires ler o IP, valida o DSCP. Se só vires MPLS, valida o bit Experimental."*

Altera a tua configuração no **Lisbon** para isto:

```bash
configure terminal

! 1. Remover o class-map antigo (que exige match-all por defeito)
no class-map CLIENTE_L2_AF

! 2. Criar novo com match-any (Critério A OU Critério B)
class-map match-any CLIENTE_L2_AF
 match ip dscp af31               ! <--- A regra do Guião
 match mpls experimental topmost 3   ! <--- A regra que vai funcionar no GNS3

! A policy-map e as interfaces já estão configuradas, não precisas mexer.
end

```

**Porquê o número 3?**
Porque no teu Wireshark vê-se claramente `Exp: 3`. O Cisco mapeou automaticamente o AF31 (que começa pelos bits 011...) para o Exp 3.

**Aplica esta alteração e faz o ping novamente.** Tenho 100% de certeza que os contadores vão subir, porque agora estamos a classificar o pacote tal como ele aparece no fio (MPLS).
